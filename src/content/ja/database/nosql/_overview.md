# NoSQL データベース知識体系

## 核心概念

NoSQL データベースはリレーショナル特性の一部を放棄し、より良いスケーラビリティと柔軟性を獲得します。

### NoSQL タイプ

```
NoSQL 分類
┌─────────────────────────────────────────┐
│  キーバリュー型 │  Redis, Memcached        │
│             │  シンプル高速、キャッシュシーン │
├─────────────┼───────────────────────────┤
│  ドキュメント型 │  MongoDB, CouchDB        │
│             │  柔軟なスキーマ、ネスト構造  │
├─────────────┼───────────────────────────┤
│  カラム指向型  │  HBase, Cassandra        │
│             │  ビッグデータ、高書き込み   │
├─────────────┼───────────────────────────┤
│  検索エンジン  │  Elasticsearch           │
│             │  全文検索、ログ分析       │
└─────────────┴───────────────────────────┘

CAP 理論
    Consistency (一貫性)
           /\
          /  \
         /    \
        /  2つ選択 \
       /________\
  Availability   Partition
  (可用性)       Tolerance
                 (分断耐性)
```

## 学習パス

### MongoDB

- **ドキュメントモデル** - BSON、ネストドキュメント
- **クエリ構文** - CRUD、集約パイプライン
- **インデックス** - 単一フィールド、複合、地理インデックス
- **クラスタ** - レプリカセット、シャーディング

### Redis

- **データ構造** - String, Hash, List, Set, ZSet
- **永続化** - RDB、AOF
- **高可用性** - マスタースレーブ、センチネル、Cluster
- **応用** - キャッシュ、分散ロック、メッセージ

### Elasticsearch

- **転置インデックス** - 全文検索原理
- **マッピング** - フィールド型定義
- **クエリ DSL** - 複雑なクエリ
- **集約** - データ分析

## ベストプラクティス

1. **選定マッチング** - シーンに合った適切なタイプを選択
2. **データモデリング** - NoSQL 特性に適応
3. **一貫性選択** - ビジネス要件に基づいてトレードオフ
4. **監視運用** - クラスタ状態監視
