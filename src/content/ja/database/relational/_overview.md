# リレーショナルデータベース知識体系

## 核心概念

リレーショナルデータベースはテーブル形式でデータを整理し、SQL でクエリし、ACID トランザクションをサポートします。

### 核心特性

```
ACID 特性
┌─────────────────────────────────────────┐
│  Atomicity    │  原子性、トランザクションは分割不可│
├───────────────┼─────────────────────────┤
│  Consistency  │  一貫性、状態遷移が正しい   │
├───────────────┼─────────────────────────┤
│  Isolation    │  分離性、トランザクションは互いに干渉しない│
├───────────────┼─────────────────────────┤
│  Durability   │  永続性、コミット後は永久に保存│
└───────────────┴─────────────────────────┘

インデックス構造
         B+ Tree インデックス
              [P]
             / | \
          [P] [P] [P]
         /|   /|   |\
       リーフノード (データポインタ)
       ↓  ↓  ↓  ↓  ↓  ↓
      順序リンク、範囲クエリ効率的
```

## 学習パス

### 1. SQL 基礎

- **DDL** - テーブル構造定義
- **DML** - データ追加・更新・削除
- **DQL** - クエリ構文、JOIN
- **DCL** - 権限制御

### 2. パフォーマンス最適化

- **インデックス最適化** - インデックス設計、実行計画
- **クエリ最適化** - SQL リライト、スロークエリ分析
- **設定チューニング** - バッファプール、接続プール

### 3. 高可用性

- **レプリケーション** - リード/ライト分離
- **クラスタ方式** - MGR、PXC
- **バックアップ/リカバリ** - 物理/論理バックアップ

## ベストプラクティス

1. **インデックス設計** - カバーリングインデックス、複合インデックス順序
2. **テーブル設計** - 適度な正規化、大きなテーブルを避ける
3. **接続プール** - 接続数を適切に設定
4. **監視** - スロークエリログ、パフォーマンスメトリクス
