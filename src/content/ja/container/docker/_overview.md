# Docker コンテナ知識体系

## 核心概念

Docker は最も人気のあるコンテナ化プラットフォームで、アプリケーションの迅速なデプロイと一貫した実行環境を実現します。

### コンテナ vs 仮想マシン

```
仮想マシンアーキテクチャ        コンテナアーキテクチャ
┌─────┬─────┬─────┐         ┌─────┬─────┬─────┐
│App A│App B│App C│         │App A│App B│App C│
├─────┼─────┼─────┤         ├─────┴─────┴─────┤
│Guest│Guest│Guest│         │  Docker Engine  │
│ OS  │ OS  │ OS  │         ├─────────────────┤
├─────┴─────┴─────┤         │    Host OS      │
│   Hypervisor    │         ├─────────────────┤
├─────────────────┤         │   Hardware      │
│    Host OS      │         └─────────────────┘
├─────────────────┤
│    Hardware     │         ✓ より軽量
└─────────────────┘         ✓ 起動が高速
                            ✓ リソース効率が高い
```

## 学習パス

### 1. 基本操作

- **イメージ管理** - pull, build, push, tag
- **コンテナライフサイクル** - run, stop, start, rm
- **リソース確認** - ps, logs, inspect, stats

### 2. イメージビルド

- **Dockerfile** - マルチステージビルド
- **イメージ最適化** - レイヤーキャッシュ、サイズ最適化
- **イメージレジストリ** - Registry, Harbor

### 3. コンテナオーケストレーション

- **Docker Compose** - マルチコンテナアプリケーション
- **ネットワークモード** - bridge, host, overlay
- **データ永続化** - volumes, bind mounts

## ベストプラクティス

1. **最小イメージ** - Alpine または distroless を使用
2. **非 root 実行** - コンテナセキュリティ向上
3. **ヘルスチェック** - HEALTHCHECK 設定
4. **ログ管理** - ログドライバーでログ収集
