# マイクロサービス知識体系

## 核心概念

マイクロサービスアーキテクチャは、アプリケーションを小さく独立してデプロイ可能なサービスに分割し、各サービスが特定のビジネス機能を中心に構築されます。

### アーキテクチャ比較

```
モノリシック vs マイクロサービス
┌─────────────────┐     ┌────┐ ┌────┐ ┌────┐
│                 │     │ユーザー│ │注文│ │決済│
│   モノリシック   │  →  │サービス│ │サービス│ │サービス│
│                 │     └────┘ └────┘ └────┘
│  ユーザー+注文+決済 │        ↓     ↓     ↓
│                 │     ┌────┐ ┌────┐ ┌────┐
└─────────────────┘     │ DB │ │ DB │ │ DB │
                        └────┘ └────┘ └────┘

マイクロサービスの特徴
┌─────────────────────────────────────────┐
│  単一責任  │  各サービスは一つのビジネスドメインに専念 │
├────────────┼────────────────────────────┤
│  独立デプロイ│  サービスは独立してリリース更新可能 │
├────────────┼────────────────────────────┤
│  技術多様性 │  異なるサービスで異なる技術スタック使用可能 │
├────────────┼────────────────────────────┤
│  障害分離  │  単一サービス障害が全体に影響しない │
└────────────┴────────────────────────────┘
```

## 学習パス

### 1. サービス設計

- **DDD** - ドメイン駆動設計
- **境界設定** - 境界付けられたコンテキスト
- **通信パターン** - 同期 vs 非同期

### 2. サービスガバナンス

- **サービスディスカバリ** - Consul, etcd, Nacos
- **ロードバランシング** - クライアント側 vs サーバー側
- **API ゲートウェイ** - 統一エントリ

### 3. フォールトトレランス設計

- **サーキットブレーカー** - Circuit Breaker
- **リトライ機構** - 指数バックオフ
- **レート制限と降格** - システム保護

## ベストプラクティス

1. **段階的分割** - モノリシックから段階的に進化
2. **自動化** - CI/CD パイプライン
3. **監視優先** - 完全な可観測性
4. **フォールトトレランス** - 障害シナリオを設計
