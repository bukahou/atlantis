# CI/CD 知識体系

## 核心概念

CI/CD は継続的インテグレーション、継続的デリバリー、継続的デプロイの実践で、ソフトウェアデリバリープロセスを自動化します。

### CI/CD プロセス

```
CI/CD パイプライン
┌──────────────────────────────────────────────────┐
│                     CI (継続的インテグレーション)   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐          │
│  │  コード  │  │  ビルド  │  │  テスト  │          │
│  │  コミット │→ │  コンパイル│→ │  検証   │          │
│  └─────────┘  └─────────┘  └─────────┘          │
└──────────────────────────────────────────────────┘
                        │
                        ▼
┌──────────────────────────────────────────────────┐
│                   CD (継続的デリバリー)            │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐          │
│  │ パッケージ│  │  デプロイ │  │  リリース │          │
│  │  イメージ │→ │ ステージング│→ │  本番   │          │
│  └─────────┘  └─────────┘  └─────────┘          │
└──────────────────────────────────────────────────┘

デプロイ戦略
┌─────────────────────────────────────────────┐
│  ローリング │  旧バージョンを段階的に置き換え    │
├────────────┼────────────────────────────────┤
│  ブルーグリーン│  2つの環境を切り替え           │
├────────────┼────────────────────────────────┤
│  カナリア   │  少量トラフィックで検証後全量     │
└────────────┴────────────────────────────────┘
```

## 学習パス

### 1. 継続的インテグレーション

- **コードチェック** - Lint、フォーマット
- **自動テスト** - ユニットテスト、統合テスト
- **ビルド成果物** - イメージ、パッケージ

### 2. 継続的デリバリー

- **環境管理** - 開発、テスト、本番
- **設定管理** - 環境変数、シークレット
- **デプロイ自動化** - スクリプト、オーケストレーション

### 3. ツールプラットフォーム

- **GitHub Actions** - クラウド CI/CD
- **Jenkins** - 自前構築 CI/CD
- **ArgoCD** - GitOps デプロイ

## ベストプラクティス

1. **高速フィードバック** - パイプラインをできるだけ早く完了
2. **環境一貫性** - コンテナ化で一貫性保証
3. **自動ロールバック** - 失敗時自動ロールバック
4. **セキュリティスキャン** - セキュリティチェック統合
