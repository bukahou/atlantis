# Git バージョン管理知識体系

## 核心概念

Git は分散バージョン管理システムで、現代ソフトウェア開発の基盤ツールです。

### Git データモデル

```
Git オブジェクトモデル
┌─────────────────────────────────────────┐
│  Commit    │  コミット、Tree と親コミットを指す│
├────────────┼────────────────────────────┤
│  Tree      │  ディレクトリ構造、Blob/Tree を含む│
├────────────┼────────────────────────────┤
│  Blob      │  ファイル内容、バイナリデータ   │
├────────────┼────────────────────────────┤
│  Tag       │  タグ、特定の Commit を指す    │
└────────────┴────────────────────────────┘

作業領域
┌──────────┐  add   ┌──────────┐ commit ┌──────────┐
│ Working  │ ────→  │ Staging  │ ────→  │  Local   │
│ Directory│        │  Area    │        │  Repo    │
└──────────┘        └──────────┘        └──────────┘
                                              │
                                              │ push
                                              ▼
                                        ┌──────────┐
                                        │  Remote  │
                                        │  Repo    │
                                        └──────────┘
```

## 学習パス

### 1. 基本操作

- **初期化** - init, clone
- **日常操作** - add, commit, push, pull
- **履歴確認** - log, diff, show

### 2. ブランチ管理

- **ブランチ操作** - branch, checkout, merge
- **リモートブランチ** - fetch, push, track
- **リベース** - rebase, cherry-pick

### 3. ワークフロー

- **Git Flow** - feature/release/hotfix
- **GitHub Flow** - 簡略化ワークフロー
- **Trunk Based** - トランクベース開発

## ベストプラクティス

1. **コミット規約** - Conventional Commits
2. **小さなコミット** - 原子化コミット
3. **コードレビュー** - PR/MR プロセス
4. **ブランチ保護** - メインブランチ保護ルール
