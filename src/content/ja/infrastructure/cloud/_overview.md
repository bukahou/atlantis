# クラウドプラットフォーム知識体系

## 核心概念

クラウドコンピューティングはソフトウェアのデリバリーと運用方法を変革しました。

### クラウドサービスモデル

```
サービス階層
┌─────────────────────────────────────┐
│           SaaS (Software as a Service) │
│       例: Gmail, Salesforce         │
├─────────────────────────────────────┤
│           PaaS (Platform as a Service) │
│    例: Heroku, Cloud Functions      │
├─────────────────────────────────────┤
│          IaaS (Infrastructure as a Service) │
│        例: EC2, GCE, Azure VM       │
└─────────────────────────────────────┘
```

## 学習パス

### 1. 核心サービス

- **コンピュートサービス** - EC2, Lambda, ECS
- **ストレージサービス** - S3, EBS, EFS
- **データベース** - RDS, DynamoDB

### 2. ネットワークとセキュリティ

- **VPC** - 仮想プライベートクラウド設定
- **IAM** - ID とアクセス管理
- **セキュリティグループ** - ネットワークアクセス制御

### 3. 高度なサービス

- **コンテナサービス** - EKS, GKE, AKS
- **サーバーレス** - Lambda, Cloud Functions
- **メッセージキュー** - SQS, SNS, Pub/Sub

## ベストプラクティス

1. **高可用性設計** - マルチ AZ デプロイ
2. **コスト最適化** - リザーブドインスタンス、Spot インスタンス
3. **セキュリティ優先** - 最小権限原則
4. **監視アラート** - 完全な可観測性の確立
