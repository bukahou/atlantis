# React 知識体系

## 核心概念

React は宣言的でコンポーネント化された JavaScript ライブラリで、Meta が開発・保守しています。

### 核心特性

```
React 核心理念
┌─────────────────────────────────────────┐
│  宣言的    │  UI 状態を記述、自動更新   │
├────────────┼───────────────────────────┤
│  コンポーネント化│  カプセル化再利用、関心の分離 │
├────────────┼───────────────────────────┤
│  単方向データフロー│  データは上から下へ、予測可能 │
├────────────┼───────────────────────────┤
│  仮想 DOM  │  効率的な更新、パフォーマンス最適化 │
└────────────┴───────────────────────────┘

コンポーネントタイプの進化
Class Component  →  Function Component + Hooks
     ↓                        ↓
状態管理が複雑          より簡潔、テストしやすい
ライフサイクルメソッド    useEffect で統一処理
```

## 学習パス

### 1. 基礎概念

- **JSX** - 構文拡張、宣言的 UI
- **コンポーネント** - 関数コンポーネント、Props 伝達
- **状態** - useState、状態の引き上げ

### 2. Hooks 深堀り

- **useState** - 状態管理
- **useEffect** - 副作用処理
- **useContext** - コンテキスト共有
- **カスタム Hooks** - ロジック再利用

### 3. 高度なパターン

- **パフォーマンス** - memo, useMemo, useCallback
- **状態管理** - Redux, Zustand, Jotai
- **SSR** - Next.js, Remix

## ベストプラクティス

1. **コンポーネント分割** - 単一責任を維持
2. **状態の下げ** - 使用箇所に近い場所に状態を配置
3. **過度な最適化を避ける** - 先に計測してから最適化
4. **TypeScript** - 型安全な開発
