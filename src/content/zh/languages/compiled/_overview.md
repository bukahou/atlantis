# 编译型语言知识体系

## 核心概念

编译型语言在运行前将源代码编译为机器码，具有更高的执行效率和更严格的类型检查。

### 编译型 vs 解释型

```
编译型语言                    解释型语言
┌──────────────┐             ┌──────────────┐
│   源代码      │             │   源代码      │
└──────┬───────┘             └──────┬───────┘
       │ 编译                        │
       ▼                             ▼
┌──────────────┐             ┌──────────────┐
│   机器码      │             │   解释器      │
└──────┬───────┘             └──────┬───────┘
       │ 执行                        │ 解释执行
       ▼                             ▼
┌──────────────┐             ┌──────────────┐
│   运行结果    │             │   运行结果    │
└──────────────┘             └──────────────┘

✓ 执行速度快                 ✓ 开发迭代快
✓ 编译时检查                 ✓ 跨平台容易
✓ 优化程度高                 ✓ 动态特性丰富
```

## 学习路径

### Go 语言

- **基础语法** - 变量、函数、结构体
- **并发编程** - Goroutine, Channel
- **标准库** - net/http, io, context
- **工程实践** - 模块化、测试、性能分析

### Rust 语言

- **所有权系统** - 借用、生命周期
- **类型系统** - trait, 泛型
- **错误处理** - Result, Option
- **并发安全** - 线程安全保证

## 语言对比

| 特性 | Go | Rust |
|-----|-----|------|
| 学习曲线 | 平缓 | 陡峭 |
| 内存管理 | GC | 所有权 |
| 并发模型 | CSP | 多范式 |
| 编译速度 | 快 | 较慢 |
| 应用场景 | 云原生/微服务 | 系统/嵌入式 |

## 最佳实践

1. **Go** - 简洁优先，善用接口
2. **Rust** - 理解所有权，拥抱编译器
3. **共同点** - 编写测试，使用 lint 工具
