# Web 语言知识体系

## 核心概念

JavaScript 是 Web 开发的核心语言，TypeScript 通过添加类型系统增强了大型项目的可维护性。

### 语言演进

```
JavaScript 演进历程
┌─────────────────────────────────────────────┐
│ ES5 (2009)    │ 严格模式、数组方法          │
├───────────────┼─────────────────────────────┤
│ ES6 (2015)    │ 类、模块、箭头函数、Promise │
├───────────────┼─────────────────────────────┤
│ ES7-ES12      │ async/await、展开运算符     │
├───────────────┼─────────────────────────────┤
│ TypeScript    │ 类型系统、接口、泛型        │
└───────────────┴─────────────────────────────┘

TypeScript 价值
┌──────────────────────────────────────────────┐
│  类型安全    →  编译时发现错误              │
│  IDE 支持    →  更好的自动补全              │
│  重构能力    →  安全的代码重构              │
│  文档作用    →  类型即文档                  │
└──────────────────────────────────────────────┘
```

## 学习路径

### JavaScript

- **基础语法** - 数据类型、函数、作用域
- **ES6+ 特性** - 解构、模块、类
- **异步编程** - Promise、async/await
- **DOM/BOM** - 浏览器 API

### TypeScript

- **类型基础** - 基本类型、联合类型
- **高级类型** - 泛型、映射类型
- **类型工具** - Partial、Pick、Omit
- **工程配置** - tsconfig、模块解析

## 技能对比

| 特性 | JavaScript | TypeScript |
|-----|-----------|------------|
| 类型系统 | 动态 | 静态 |
| 编译步骤 | 不需要 | 需要 |
| IDE 支持 | 基础 | 完善 |
| 学习曲线 | 较低 | 中等 |
| 适用项目 | 小型/快速迭代 | 大型/团队协作 |

## 最佳实践

1. **TypeScript** - 严格模式、避免 any
2. **异步处理** - 优先 async/await
3. **模块化** - ES Modules 规范
4. **代码质量** - ESLint + Prettier
