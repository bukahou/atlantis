# 关系型数据库知识体系

## 核心概念

关系型数据库以表格形式组织数据，通过 SQL 进行查询，支持 ACID 事务。

### 核心特性

```
ACID 特性
┌─────────────────────────────────────────┐
│  Atomicity    │  原子性，事务不可分割   │
├───────────────┼─────────────────────────┤
│  Consistency  │  一致性，状态转换正确   │
├───────────────┼─────────────────────────┤
│  Isolation    │  隔离性，事务互不干扰   │
├───────────────┼─────────────────────────┤
│  Durability   │  持久性，提交后永久保存 │
└───────────────┴─────────────────────────┘

索引结构
         B+ Tree 索引
              [P]
             / | \
          [P] [P] [P]
         /|   /|   |\
       叶子节点 (数据指针)
       ↓  ↓  ↓  ↓  ↓  ↓
      顺序链接，范围查询高效
```

## 学习路径

### 1. SQL 基础

- **DDL** - 表结构定义
- **DML** - 数据增删改
- **DQL** - 查询语法、JOIN
- **DCL** - 权限控制

### 2. 性能优化

- **索引优化** - 索引设计、执行计划
- **查询优化** - SQL 重写、慢查询分析
- **配置调优** - 缓冲池、连接池

### 3. 高可用

- **主从复制** - 读写分离
- **集群方案** - MGR、PXC
- **备份恢复** - 物理备份、逻辑备份

## 数据库对比

| 特性 | MySQL | PostgreSQL |
|-----|-------|------------|
| 性能 | 读取快 | 复杂查询强 |
| 扩展性 | 插件化 | 丰富的类型 |
| JSON | 支持 | 原生支持 |
| 社区 | 大 | 活跃 |
| 适用 | Web 应用 | 复杂业务 |

## 最佳实践

1. **索引设计** - 覆盖索引、联合索引顺序
2. **表设计** - 适度规范化，避免大表
3. **连接池** - 合理配置连接数
4. **监控** - 慢查询日志、性能指标
