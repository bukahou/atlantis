# 云平台知识体系

## 核心概念

云计算改变了软件交付和运维方式，理解云服务模型和核心服务对于现代架构设计至关重要。

### 云服务模型

```
服务层次
┌─────────────────────────────────────┐
│           SaaS (软件即服务)          │
│       例: Gmail, Salesforce         │
├─────────────────────────────────────┤
│           PaaS (平台即服务)          │
│    例: Heroku, Cloud Functions      │
├─────────────────────────────────────┤
│          IaaS (基础设施即服务)        │
│        例: EC2, GCE, Azure VM       │
└─────────────────────────────────────┘

责任划分
┌──────────┬──────────┬──────────┐
│   IaaS   │   PaaS   │   SaaS   │
├──────────┼──────────┼──────────┤
│ 应用 ←用户│ 应用 ←用户│ 全部由   │
│ 数据 ←用户│ 数据 ←用户│ 云厂商   │
│ 运行时←用户│ 运行时←云 │ 管理     │
│ 中间件←用户│ 中间件←云 │          │
│ OS   ←用户│ OS   ←云 │          │
│ 虚拟化←云 │ 虚拟化←云 │          │
│ 物理 ←云 │ 物理 ←云 │          │
└──────────┴──────────┴──────────┘
```

## 学习路径

### 1. 核心服务

- **计算服务** - EC2, Lambda, ECS
- **存储服务** - S3, EBS, EFS
- **数据库** - RDS, DynamoDB, ElastiCache

### 2. 网络与安全

- **VPC** - 虚拟私有云配置
- **IAM** - 身份与访问管理
- **安全组** - 网络访问控制

### 3. 高级服务

- **容器服务** - EKS, GKE, AKS
- **Serverless** - Lambda, Cloud Functions
- **消息队列** - SQS, SNS, Pub/Sub

## 主流云平台对比

| 服务类型 | AWS | GCP | Azure |
|---------|-----|-----|-------|
| 计算 | EC2 | GCE | VM |
| 容器 | EKS | GKE | AKS |
| 对象存储 | S3 | GCS | Blob |
| 数据库 | RDS | Cloud SQL | SQL DB |
| Serverless | Lambda | Functions | Functions |

## 最佳实践

1. **高可用设计** - 多可用区部署
2. **成本优化** - 使用预留实例、Spot 实例
3. **安全优先** - 最小权限原则
4. **监控告警** - 建立完善的可观测性
