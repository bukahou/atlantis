# Linux 系统知识体系

## 核心概念

Linux 是服务器端最主流的操作系统，掌握 Linux 是后端开发和运维的必备技能。

### 系统架构

```
用户空间 (User Space)
┌────────────────────────────────────┐
│  应用程序  │  Shell  │  系统工具   │
├────────────────────────────────────┤
│           C 标准库 (glibc)         │
├────────────────────────────────────┤
│           系统调用接口             │
└────────────────────────────────────┘
─────────────────────────────────────
内核空间 (Kernel Space)
┌────────────────────────────────────┐
│  进程调度 │ 内存管理 │ 文件系统    │
├────────────────────────────────────┤
│  网络栈   │ 设备驱动 │ 安全模块    │
└────────────────────────────────────┘
```

## 学习路径

### 1. 基础操作

- **文件管理** - ls, cp, mv, rm, find
- **权限控制** - chmod, chown, ACL
- **用户管理** - useradd, usermod, groups

### 2. 系统管理

- **进程管理** - ps, top, kill, systemctl
- **内存管理** - free, vmstat, /proc/meminfo
- **磁盘管理** - df, du, fdisk, mount

### 3. 高级运维

- **性能分析** - perf, strace, sar
- **日志管理** - journalctl, rsyslog
- **自动化** - cron, systemd timer

## 关键技能

| 技能领域 | 核心内容 | 应用场景 |
|---------|---------|---------|
| 命令行 | Bash, Zsh | 日常操作 |
| 编辑器 | Vim, Nano | 配置修改 |
| 包管理 | apt, yum, dnf | 软件安装 |
| 网络配置 | ip, ss, netstat | 网络运维 |

## 最佳实践

1. **最小权限** - 避免使用 root，善用 sudo
2. **日志分析** - 定期检查系统日志
3. **备份策略** - 关键数据定期备份
4. **安全加固** - 关闭不必要服务，配置防火墙
