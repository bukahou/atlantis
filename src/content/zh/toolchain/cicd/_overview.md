# CI/CD 知识体系

## 核心概念

CI/CD 是持续集成、持续交付和持续部署的实践，自动化软件交付流程。

### CI/CD 流程

```
CI/CD 流水线
┌──────────────────────────────────────────────────┐
│                     CI (持续集成)                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐          │
│  │  代码   │  │  构建   │  │  测试   │          │
│  │  提交   │→ │  编译   │→ │  验证   │          │
│  └─────────┘  └─────────┘  └─────────┘          │
└──────────────────────────────────────────────────┘
                        │
                        ▼
┌──────────────────────────────────────────────────┐
│                   CD (持续交付)                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐          │
│  │  打包   │  │  部署   │  │  发布   │          │
│  │  镜像   │→ │  预发布 │→ │  生产   │          │
│  └─────────┘  └─────────┘  └─────────┘          │
└──────────────────────────────────────────────────┘

部署策略
┌─────────────────────────────────────────────┐
│  滚动更新  │  逐步替换旧版本              │
├────────────┼────────────────────────────────┤
│  蓝绿部署  │  两套环境切换                │
├────────────┼────────────────────────────────┤
│  金丝雀    │  小流量验证后全量            │
└────────────┴────────────────────────────────┘
```

## 学习路径

### 1. 持续集成

- **代码检查** - Lint、格式化
- **自动测试** - 单元测试、集成测试
- **构建产物** - 镜像、包

### 2. 持续交付

- **环境管理** - 开发、测试、生产
- **配置管理** - 环境变量、密钥
- **部署自动化** - 脚本、编排

### 3. 工具平台

- **GitHub Actions** - 云端 CI/CD
- **Jenkins** - 自建 CI/CD
- **ArgoCD** - GitOps 部署

## 工具对比

| 工具 | 特点 | 适用场景 |
|-----|------|---------|
| GitHub Actions | 云原生、易用 | GitHub 项目 |
| Jenkins | 功能强大、可扩展 | 企业自建 |
| GitLab CI | 集成度高 | GitLab 生态 |
| ArgoCD | GitOps | Kubernetes |

## 最佳实践

1. **快速反馈** - 流水线尽快完成
2. **环境一致** - 容器化保证一致性
3. **自动回滚** - 失败自动回滚
4. **安全扫描** - 集成安全检查
