# React 知识体系

## 核心概念

React 是构建用户界面的声明式、组件化 JavaScript 库，由 Meta 开发维护。

### 核心特性

```
React 核心理念
┌─────────────────────────────────────────┐
│  声明式     │  描述 UI 状态，自动更新   │
├─────────────┼───────────────────────────┤
│  组件化     │  封装复用，关注点分离     │
├─────────────┼───────────────────────────┤
│  单向数据流 │  数据自上而下，可预测     │
├─────────────┼───────────────────────────┤
│  虚拟 DOM   │  高效更新，性能优化       │
└─────────────┴───────────────────────────┘

组件类型演进
Class Component  →  Function Component + Hooks
     ↓                        ↓
状态管理复杂           更简洁、更易测试
生命周期方法           useEffect 统一处理
```

## 学习路径

### 1. 基础概念

- **JSX** - 语法扩展，声明式 UI
- **组件** - 函数组件、Props 传递
- **状态** - useState、状态提升

### 2. Hooks 深入

- **useState** - 状态管理
- **useEffect** - 副作用处理
- **useContext** - 上下文共享
- **自定义 Hooks** - 逻辑复用

### 3. 高级模式

- **性能优化** - memo, useMemo, useCallback
- **状态管理** - Redux, Zustand, Jotai
- **服务端渲染** - Next.js, Remix

## 关键技能

| 技能领域 | 核心内容 | 应用场景 |
|---------|---------|---------|
| 组件设计 | 高阶组件、Render Props | 代码复用 |
| 状态管理 | Context、Redux | 复杂状态 |
| 性能优化 | 懒加载、代码分割 | 大型应用 |
| 测试 | Jest、React Testing Library | 质量保障 |

## 最佳实践

1. **组件拆分** - 保持组件单一职责
2. **状态下沉** - 状态尽量靠近使用处
3. **避免过度优化** - 先测量再优化
4. **TypeScript** - 类型安全开发
